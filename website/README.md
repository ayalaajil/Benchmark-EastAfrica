# Verification results dashboard

A self-contained static website (no build step, no server, no dependencies)
presenting every figure and table produced by `run_verification.py` for the
MAM 2024 benchmark. UI modeled on
[RainCheck Africa](https://africlimate-research.github.io/RainCheckAfrica/):
header with headline stats, tab views, filter chips, lightbox zoom.

## View it

Open `index.html` directly in a browser (works from `file://` — all table data
is embedded, nothing is fetched), or serve it:

```bash
python -m http.server 8000 --directory website
# → http://localhost:8000
```

## Refresh after a pipeline rerun

```bash
python run_verification.py          # regenerates mam2024_analysis_outputs/
python website/sync_outputs.py      # mirrors figures/CSVs here + re-embeds tables
```

`sync_outputs.py` cross-validates `manifest.js` against the output folder in
both directions and **fails** if they disagree (renamed/missing/unlisted
files), so the site can't silently go stale. If the pipeline gains or renames
a figure, add/update its entry in `manifest.js`.

## Layout

```
index.html        page skeleton (tabs, panels, lightbox, about)
style.css         design system — palette mirrors benchmark_ea/verification/style.py
app.js            tabs, filters, lightbox, sortable tables (vanilla JS)
manifest.js       hand-authored catalog: figure titles/captions/tabs/tags, table list
sync_outputs.py   mirror + validate + embed (run after every pipeline rerun)
figures/          PNGs + PDF twins   (generated, committed)
tables/           raw CSVs           (generated, committed)
data/tables.js    embedded CSV data  (generated, committed)
```

`figures/`, `tables/` and `data/tables.js` are generated by the sync script
but committed, so the site is deployable as-is.

## Deployment

Deployed automatically by `.github/workflows/docs.yml` on every push to `main`
touching `website/**` (or the docs): the workflow builds the MkDocs site,
copies this folder into it as `site/dashboard/`, and publishes the combined
tree to `gh-pages`. Live URLs:

- docs: https://ayalaajil.github.io/Benchmark-EastAfrica/
- dashboard: https://ayalaajil.github.io/Benchmark-EastAfrica/dashboard/

Don't push to `gh-pages` directly — the workflow force-overwrites it.
